GRUB2 Rescue 模式时使用的开机命令

使用 Grub2 作为开机选单时，若设置错误或安装失败，而在开机时不显示启动选单，而是显示 grub-rescue> 这一提示符。那就需要使用者手动输入开机命令来启动操作系统。
或者 Grub2 选单正常时，也可以按照屏幕下方的提示，按下 c 来进入命令列模式，此时可以看到 grub> 提示符。

以下各命令列，均是启动 Linux 操作系统
以下各命令列，输完每一行都按下 Enter
输入命令或路径时，可以按下 Tab 来补全
注意在救援模式时，即提示符为 grub-rescue> 时，需要载入 linux 模组，否则会提示“Unknown command linux”

首先查看下当前的 Grub2 设定：

set    # 仅输入 set 而不加设置对象时，此命令显示当前的 Grub2 设置
ls    # 使用 ls 来查看 Grub2 所认出的装置。例如：(hd0) = sda; (hd0,1)=sda1; (hd0,msdos2)=sda2

以最近的核心快速开机：

set root=(hdX,Y)    # 输入来自 ls 指令查出的正确的 X,Y 值，例如：set root=(hd0,msdos1)
insmod /boot/grub/linux.mod    # 该行命令仅在提示符为 grub-rescue> 时需要,用于载入 linux 模组，若没有载入此模组，将会提示 “Unknown command linux”
linux /vmlinuz root=/dev/sdXY ro   # 接上例，linux /vmlinuz root=/dev/sda1 ro
initrd /initrd.img    # 选择最新的 initrd 镜像
boot    # 以上都成功设定好以后，输入 boot 启动该操作系统

手动开机至指定核心：

set root=(hdX,Y)    # 输入来自 ls 指令查出的正确的 X,Y 值，例如：set root=(hd0,msdos1)
insmod /boot/grub/linux.mod    # 该行命令仅在提示符为 grub-rescue> 时需要,用于载入 linux 模组，若没有载入此模组，将会提示 “Unknown command linux”
linux /boot/vmlinuz-3.2.0-67-generic root=/dev/sda1 ro    # 该行命令由四部分组成：1.linux 2.kernel 3.root 4.ro； 在输入 kernel 时，可以先输到 /boot/vml 的“l”然后紧接着按下 Tab 键来自动补全 kernel 的路径； root的路径对应上一步骤：root=/dev/sda1；最后输入的 ro 为加入的选项
initrd /boot/initrd.img-3.2.0-67-generic    # 选择 initrd 镜像，同样在输入路径时，可以输到 /boot/ini 的“i”后，紧接着按下 Tab 来补全路径
boot    # 以上都成功设定好以后，输入 boot 启动该操作系统
